# Security policy

## Supported versions

Security fixes are applied to the latest release of spotr.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's
private vulnerability reporting feature on the Security tab of this repository.
Include the affected version, reproduction steps, impact, and any suggested
mitigation. You should receive an initial response within seven days.

spotr is local-first and does not send workout data to a hosted service. A
report involving unexpected file access, unsafe template handling, release
artifact integrity, or dependency compromise is still in scope.
